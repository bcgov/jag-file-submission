# Input

Text inputs allow users to enter a single line of text.

## When to Use

- Input text is no longer than a single line
- Users have to input an answer that cannot be easily predicted

## When not to Use

- A user needs to input multiple lines of text
- Users can only choose from a selected list of responses

## API

| Prop     | Type     | Required | Description                                           | Default |
| -------- | -------- | -------- | ----------------------------------------------------- | ------- |
| input    | object   | true     | object with input field details                       |         |
| onChange | function | true     | function to execute when value in input field changes |         |

### Input API

The input object has its own properties that need to be present within it, as shown below:

| Prop        | Type    | Required | Description                                                                                    | Default |
| ----------- | ------- | -------- | ---------------------------------------------------------------------------------------------- | ------- |
| label       | string  | false    | value of the heading label for the input field                                                 |         |
| id          | string  | true     | value of the unique identifier for the input field                                             |         |
| value       | string  | false    | actual value present in the input field                                                        |         |
| placeholder | string  | false    | placeholder string present in the input field                                                  |         |
| isRequired  | boolean | true     | value representing whether the input field is mandatory or not                                 |         |
| errorMsg    | string  | false    | value to be displayed as an error message if the value entered in the input field is incorrect |         |
| styling     | string  | true     | css classes to apply styling to the input field                                                |         |
| note        | string  | false    | note for the label of the input field                                                          |         |
| isReadOnly  | boolean | false    | if true, the input field is non-editable                                                       |         |

## Editable White With Label

The most commonly used input field with a label heading.

<Story
  name="Editable White With Label"
  id="input--editable-white-with-label"
></Story>

```bash
const input = {
  label: "Label Heading",
  id: "textInputId",
  placeholder: "Enter value",
  isReadOnly: false,
  isRequired: false
};

<Input
  input={{
    ...input,
    styling: "editable-white"
  }}
  onChange={action("onChange")}
/>
```
